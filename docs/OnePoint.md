**这个项目是做什么的**
```aiignore
一个本地社区生活服务平台，提供商户优惠券秒杀、用户管理、私信聊天、博客评价、商铺分类与距离查询等功能。
```
**项目价值/亮点**
```aiignore
一个高并发秒杀系统的完整优化案例
通过个人改造，系统在单机虚拟机环境下可支撑2000+并发秒杀请求,所有到达服务端的请求均返回成功，7.89%的错误来自JMeter与虚拟机之间的TCP连接损耗，非业务逻辑错误。
```

**负责什么**
```aiignore
独立完成秒杀流程设计和搭建：Redis分布式锁+Lua脚本原子扣库存+RabbitMQ异步下单+死信队列异常兜底
店铺查询，用户私信聊天，博客系统评价和发布
容器化部署:Docker Compose 一键编排MySQL+Redis+RabbitMQ+Nginx+应用服务
```

**最核心的改造**
```aiignore
最核心的改造是秒杀链路的完善,在原有分布式锁和lua脚本原则性扣除库存的基础上
添加了滑动窗口限流，防止外部恶意访问
添加了消息队列，进行削峰填谷，异步完成订单生成，并添加轮询接口对订单进行查询，同时死信队列对异常消息进行处理和记录，以便后续进行人工介入
```

**遇到了什么困难？怎么解决的？**
```aiignore
RabbitMQ配置类中未对消息对象配置序列化器，导致发送消息时出现序列化错误。在配置类中加入了Jackson2JsonMessageConverter来解决。

Lua脚本中未对数据进行tonumber()显式转换，导致库存扣减逻辑判断异常。在脚本中对数据增加了tonumber()封装以修正

关于压测100%报错排查原因:Lua脚本中INCREBY对脏数据敏感导致奔溃，改成SET命令根治

连接池瓶颈：Redis max-active从10->400，Tomcat线程池从200->400，并添加了线程池缓存，逐轮调优验证
```
